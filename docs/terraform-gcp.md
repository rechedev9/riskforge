## 1. Provider Setup

### Basic Provider Configuration

```hcl
terraform {
  required_version = ">= 1.5"

  required_providers {
    google = {
      source  = "hashicorp/google"
      version = "~> 6.0"
    }
    google-beta = {
      source  = "hashicorp/google-beta"
      version = "~> 6.0"
    }
  }
}

provider "google" {
  project = var.project_id
  region  = var.region
}

provider "google-beta" {
  project = var.project_id
  region  = var.region
}
```

### Authentication Methods (Ordered by Preference)

**1. Workload Identity Federation (CI/CD -- preferred)**

No credentials in config. The CI/CD pipeline obtains short-lived tokens via OIDC.
Set `GOOGLE_APPLICATION_CREDENTIALS` to point at a credential configuration file
generated by the WIF auth action. See [Section 11: CI/CD](#11-cicd).

**2. Service Account Impersonation (shared environments)**

```hcl
provider "google" {
  project                     = var.project_id
  region                      = var.region
  impersonate_service_account = "terraform@${var.project_id}.iam.gserviceaccount.com"
}
```

Requires the caller to have `roles/iam.serviceAccountTokenCreator` on the target SA.

**3. Application Default Credentials (local dev)**

```bash
gcloud auth application-default login
```

The provider picks up credentials automatically -- no config needed.

**4. Service Account Key (discouraged)**

```bash
export GOOGLE_APPLICATION_CREDENTIALS="/path/to/key.json"
```

Avoid committing key files. Prefer WIF or impersonation.

### Provider Configuration Reference

| Argument                          | Description                                    |
|-----------------------------------|------------------------------------------------|
| `project`                         | Default GCP project ID                         |
| `region`                          | Default region (e.g., `us-central1`)           |
| `zone`                            | Default zone (e.g., `us-central1-a`)           |
| `credentials`                     | Path to JSON key file or JSON contents         |
| `access_token`                    | OAuth 2.0 bearer token                         |
| `impersonate_service_account`     | SA email to impersonate                        |
| `scopes`                          | OAuth scopes list                              |
| `default_labels`                  | Labels applied to all resources                |

## 2. State Management

### GCS Backend for Remote State

```hcl
terraform {
  backend "gcs" {
    bucket = "my-project-terraform-state"
    prefix = "environments/prod"
  }
}
```

State locking is built-in with the GCS backend -- no DynamoDB equivalent needed.

### State Bucket Configuration (bootstrap)

```hcl
resource "google_storage_bucket" "terraform_state" {
  name     = "${var.project_id}-terraform-state"
  location = var.region
  project  = var.project_id

  # Prevent accidental deletion
  force_destroy = false

  versioning {
    enabled = true
  }

  # Clean up old state versions after 30 days
  lifecycle_rule {
    condition {
      num_newer_versions = 5
    }
    action {
      type = "Delete"
    }
  }

  uniform_bucket_level_access = true

  labels = {
    purpose = "terraform-state"
  }
}
```

### Backend Arguments Reference

| Argument                              | Env Var                                    | Purpose                          |
|---------------------------------------|--------------------------------------------|----------------------------------|
| `bucket` (required)                   | --                                         | Globally unique bucket name      |
| `prefix`                              | --                                         | Object path prefix               |
| `credentials`                         | `GOOGLE_BACKEND_CREDENTIALS`               | Path to SA key JSON              |
| `impersonate_service_account`         | `GOOGLE_BACKEND_IMPERSONATE_SERVICE_ACCOUNT` | SA to impersonate              |
| `encryption_key`                      | `GOOGLE_ENCRYPTION_KEY`                    | 32-byte base64 CSEK             |
| `kms_encryption_key`                  | `GOOGLE_KMS_ENCRYPTION_KEY`                | Cloud KMS key resource path      |

## 3. Cloud Run

### google_cloud_run_v2_service -- Full Configuration

```hcl
resource "google_cloud_run_v2_service" "api" {
  name                = "api-service"
  location            = var.region
  ingress             = "INGRESS_TRAFFIC_ALL"
  deletion_protection = true

  template {
    service_account = google_service_account.cloud_run_sa.email

    scaling {
      min_instance_count = var.min_instances  # 0 for scale-to-zero
      max_instance_count = var.max_instances  # e.g., 10
    }

    containers {
      image = "${var.region}-docker.pkg.dev/${var.project_id}/${var.repo}/${var.image}:${var.image_tag}"

      ports {
        container_port = 8080
      }

      resources {
        limits = {
          cpu    = "2"
          memory = "1024Mi"
        }
        cpu_idle         = true   # Allocate CPU only during requests
        startup_cpu_boost = true  # Extra CPU at cold start
      }

      # Plain environment variables
      env {
        name  = "PROJECT_ID"
        value = var.project_id
      }
      env {
        name  = "SPANNER_INSTANCE"
        value = var.spanner_instance
      }

      # Secret from Secret Manager
      env {
        name = "DATABASE_URL"
        value_source {
          secret_key_ref {
            secret  = google_secret_manager_secret.db_url.secret_id
            version = "latest"
          }
        }
      }

      # Volume-mounted secrets
      volume_mounts {
        name       = "service-creds"
        mount_path = "/secrets"
      }

      # Health checks
      startup_probe {
        http_get {
          path = "/health"
          port = 8080
        }
        initial_delay_seconds = 5
        period_seconds        = 3
        failure_threshold     = 3
      }

      liveness_probe {
        http_get {
          path = "/health"
          port = 8080
        }
        period_seconds = 30
      }
    }

    # Secret volumes
    volumes {
      name = "service-creds"
      secret {
        secret       = google_secret_manager_secret.service_creds.secret_id
        default_mode = 292  # 0444
        items {
          version = "latest"
          path    = "credentials.json"
        }
      }
    }

    # VPC access for private resources
    vpc_access {
      connector = google_vpc_access_connector.connector.id
      egress    = "PRIVATE_RANGES_ONLY"
    }

    timeout                          = "300s"
    max_instance_request_concurrency = 80
    execution_environment            = "EXECUTION_ENVIRONMENT_GEN2"
  }

  # Traffic routing
  traffic {
    type    = "TRAFFIC_TARGET_ALLOCATION_TYPE_LATEST"
    percent = 100
  }

  labels = var.labels
}
```

### Traffic Splitting (Canary Deployments)

```hcl
resource "google_cloud_run_v2_service" "api" {
  name     = "api-service"
  location = var.region

  template {
    revision = "api-service-v2"
    containers {
      image = "us-docker.pkg.dev/myproject/repo/api:v2"
    }
  }

  # 90% stable, 10% canary
  traffic {
    type     = "TRAFFIC_TARGET_ALLOCATION_TYPE_REVISION"
    revision = "api-service-v1"
    percent  = 90
  }
  traffic {
    type     = "TRAFFIC_TARGET_ALLOCATION_TYPE_REVISION"
    revision = "api-service-v2"
    percent  = 10
    tag      = "canary"
  }
}
```

### Database with DDL

```hcl
resource "google_spanner_database" "main" {
  instance = google_spanner_instance.main.name
  name     = "main-database"

  version_retention_period = "3d"  # Point-in-time recovery window (1h to 7d)
  database_dialect         = "GOOGLE_STANDARD_SQL"
  deletion_protection      = true
  enable_drop_protection   = true

  ddl = [
    <<-EOT
      CREATE TABLE Policies (
        PolicyId     STRING(36) NOT NULL,
        CustomerId   STRING(36) NOT NULL,
        PolicyNumber STRING(20) NOT NULL,
        Type         STRING(50) NOT NULL,
        Status       STRING(20) NOT NULL,
        Premium      FLOAT64 NOT NULL,
        CreatedAt    TIMESTAMP NOT NULL OPTIONS (allow_commit_timestamp=true),
        UpdatedAt    TIMESTAMP NOT NULL OPTIONS (allow_commit_timestamp=true),
      ) PRIMARY KEY(PolicyId)
    EOT
    ,
    <<-EOT
      CREATE TABLE Documents (
        DocumentId STRING(36) NOT NULL,
        PolicyId   STRING(36) NOT NULL,
        FileName   STRING(256) NOT NULL,
        GcsUri     STRING(1024) NOT NULL,
        CreatedAt  TIMESTAMP NOT NULL OPTIONS (allow_commit_timestamp=true),
      ) PRIMARY KEY(PolicyId, DocumentId),
        INTERLEAVE IN PARENT Policies ON DELETE CASCADE
    EOT
    ,
    "CREATE INDEX PolicyByCustomer ON Policies(CustomerId)",
    "CREATE INDEX PolicyByNumber ON Policies(PolicyNumber)",
  ]
}
```

### Push Subscription with OIDC (Cloud Run target)

```hcl
resource "google_pubsub_subscription" "policy_events_push" {
  name  = "policy-events-push"
  topic = google_pubsub_topic.policy_events.id

  ack_deadline_seconds       = 20
  message_retention_duration = "604800s"  # 7 days

  push_config {
    push_endpoint = "${google_cloud_run_v2_service.api.uri}/webhooks/pubsub"

    # OIDC token for authenticated push to Cloud Run
    oidc_token {
      service_account_email = google_service_account.pubsub_sa.email
      audience              = google_cloud_run_v2_service.api.uri
    }

    attributes = {
      x-goog-version = "v1"
    }
  }

  # Dead letter policy
  dead_letter_policy {
    dead_letter_topic     = google_pubsub_topic.policy_events_dlq.id
    max_delivery_attempts = 10
  }

  # Retry policy with exponential backoff
  retry_policy {
    minimum_backoff = "10s"
    maximum_backoff = "600s"
  }

  # Subscription never auto-deletes
  expiration_policy {
    ttl = ""  # empty string = never expires
  }

  labels = var.labels
}
```

### IAM Best Practices

| Pattern                          | Resource                                | Notes                                             |
|----------------------------------|-----------------------------------------|---------------------------------------------------|
| Additive binding                 | `google_project_iam_member`             | Preferred -- does not clobber other bindings       |
| Authoritative per-role binding   | `google_project_iam_binding`            | Replaces all members for that role                 |
| Authoritative full policy        | `google_project_iam_policy`             | Dangerous -- replaces the entire project policy    |
| Resource-level binding           | `google_*_iam_member`                   | Scope to specific resource (bucket, topic, etc.)   |
| Custom role                      | `google_project_iam_custom_role`        | Fine-grained permissions beyond predefined roles   |

Source: [Workload Identity Federation](https://docs.cloud.google.com/iam/docs/workload-identity-federation-with-deployment-pipelines) |
[google_iam_workload_identity_pool_provider](https://registry.terraform.io/providers/hashicorp/google/latest/docs/resources/iam_workload_identity_pool_provider)

    condition {
      age = 30
    }

      storage_class = "NEARLINE"

      age = 90
    }
    action {
      type          = "SetStorageClass"
      storage_class = "COLDLINE"
    }
  }

  # Delete after 7 years (regulatory retention)
  lifecycle_rule {
    condition {
      age = 2555  # ~7 years
    }
    action {
      type = "Delete"
    }

  # Clean up old versions
  lifecycle_rule {
    condition {
      num_newer_versions = 3
      with_state         = "ARCHIVED"
    }
    action {
      type = "Delete"
    }
  }

  # Abort incomplete multipart uploads after 7 days
  lifecycle_rule {
    condition {
      age = 7

    action {
      type = "AbortIncompleteMultipartUpload"

    origin          = var.cors_origins

## 11. CI/CD

### GitHub Actions Workflow with Workload Identity Federation

```yaml
# .github/workflows/terraform.yml
name: Terraform

on:
  pull_request:
    branches: [main]
    paths:
      - 'infrastructure/**'
      - '.github/workflows/terraform.yml'
  push:
    branches: [main]
    paths:
      - 'infrastructure/**'
      - '.github/workflows/terraform.yml'

permissions:
  contents: read
  pull-requests: write
  id-token: write  # Required for WIF OIDC token

env:
  TF_VERSION: '1.9'
  WORKING_DIR: 'infrastructure/environments/prod'
  WIF_PROVIDER: 'projects/PROJECT_NUMBER/locations/global/workloadIdentityPools/github-actions/providers/github-provider'
  SERVICE_ACCOUNT: 'terraform@PROJECT_ID.iam.gserviceaccount.com'

jobs:
  terraform-plan:
    name: Terraform Plan
    runs-on: ubuntu-latest
    if: github.event_name == 'pull_request'

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Authenticate to Google Cloud
        id: auth
        uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ env.WIF_PROVIDER }}
          service_account: ${{ env.SERVICE_ACCOUNT }}
          create_credentials_file: true

      - name: Setup Terraform
        uses: hashicorp/setup-terraform@v3
        with:
          terraform_version: ${{ env.TF_VERSION }}

      - name: Terraform Init
        working-directory: ${{ env.WORKING_DIR }}
        run: terraform init -input=false

      - name: Terraform Validate
        working-directory: ${{ env.WORKING_DIR }}
        run: terraform validate

      - name: Terraform Plan
        id: plan
        working-directory: ${{ env.WORKING_DIR }}
        run: terraform plan -input=false -no-color -out=tfplan
        continue-on-error: true

      - name: Comment Plan on PR
        uses: actions/github-script@v7
        if: github.event_name == 'pull_request'
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}
          script: |
            const output = `#### Terraform Plan \`${{ steps.plan.outcome }}\`

            <details><summary>Show Plan</summary>

            \`\`\`terraform
            ${{ steps.plan.outputs.stdout }}
            \`\`\`

            </details>

            *Pushed by: @${{ github.actor }}*`;

            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: output
            });

      - name: Plan Status
        if: steps.plan.outcome == 'failure'
        run: exit 1

  terraform-apply:
    name: Terraform Apply
    runs-on: ubuntu-latest
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    environment: production  # Requires GitHub environment approval

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Authenticate to Google Cloud
        id: auth
        uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ env.WIF_PROVIDER }}
          service_account: ${{ env.SERVICE_ACCOUNT }}
          create_credentials_file: true

      - name: Setup Terraform
        uses: hashicorp/setup-terraform@v3
        with:
          terraform_version: ${{ env.TF_VERSION }}

      - name: Terraform Init
        working-directory: ${{ env.WORKING_DIR }}
        run: terraform init -input=false

      - name: Terraform Apply
        working-directory: ${{ env.WORKING_DIR }}
        run: terraform apply -input=false -auto-approve
```

### Multi-Environment Workflow

```yaml
# .github/workflows/terraform-multi-env.yml
name: Terraform Multi-Environment

on:
  pull_request:
    branches: [main]
    paths: ['infrastructure/**']
  push:
    branches: [main]
    paths: ['infrastructure/**']

permissions:
  contents: read
  pull-requests: write
  id-token: write

jobs:
  plan:
    name: Plan (${{ matrix.environment }})
    runs-on: ubuntu-latest
    if: github.event_name == 'pull_request'
    strategy:
      matrix:
        environment: [dev, prod]
      fail-fast: false

    steps:
      - uses: actions/checkout@v4

      - uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ vars.WIF_PROVIDER }}
          service_account: ${{ vars.TERRAFORM_SA }}

      - uses: hashicorp/setup-terraform@v3
        with:
          terraform_version: '1.9'

      - name: Init & Plan
        working-directory: infrastructure/environments/${{ matrix.environment }}
        run: |
          terraform init -input=false
          terraform plan -input=false -no-color

  deploy-dev:
    name: Deploy Dev
    runs-on: ubuntu-latest
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    environment: dev

    steps:
      - uses: actions/checkout@v4

      - uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ vars.WIF_PROVIDER }}
          service_account: ${{ vars.TERRAFORM_SA }}

      - uses: hashicorp/setup-terraform@v3
        with:
          terraform_version: '1.9'

      - name: Init & Apply
        working-directory: infrastructure/environments/dev
        run: |
          terraform init -input=false
          terraform apply -input=false -auto-approve

  deploy-prod:
    name: Deploy Prod
    runs-on: ubuntu-latest
    needs: deploy-dev
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    environment: production  # Manual approval gate

    steps:
      - uses: actions/checkout@v4

      - uses: google-github-actions/auth@v2
        with:
          workload_identity_provider: ${{ vars.WIF_PROVIDER }}
          service_account: ${{ vars.TERRAFORM_SA }}

      - uses: hashicorp/setup-terraform@v3
        with:
          terraform_version: '1.9'

      - name: Init & Apply
        working-directory: infrastructure/environments/prod
        run: |
          terraform init -input=false
          terraform apply -input=false -auto-approve
```

### WIF Bootstrap (Terraform for the Terraform SA)

This bootstraps the Workload Identity Federation pool and provider that the CI/CD
pipeline uses. Run this manually once (or from a separate bootstrap pipeline with
a privileged SA).

```hcl
# infrastructure/bootstrap/main.tf

locals {
  project_number = data.google_project.current.number
}

data "google_project" "current" {
  project_id = var.project_id
}

# Service account for Terraform automation
resource "google_service_account" "terraform" {
  account_id   = "terraform"
  display_name = "Terraform CI/CD"
  project      = var.project_id
}

# Permissions for Terraform SA (scope to what Terraform needs to manage)
resource "google_project_iam_member" "terraform_roles" {
  for_each = toset([
    "roles/run.admin",
    "roles/spanner.admin",
    "roles/pubsub.admin",
    "roles/storage.admin",
    "roles/secretmanager.admin",
    "roles/monitoring.admin",
    "roles/vpcaccess.admin",
    "roles/iam.serviceAccountAdmin",
    "roles/iam.serviceAccountUser",
    "roles/iam.workloadIdentityPoolAdmin",
    "roles/resourcemanager.projectIamAdmin",
  ])

  project = var.project_id
  role    = each.key
  member  = "serviceAccount:${google_service_account.terraform.email}"
}

# Workload Identity Pool
resource "google_iam_workload_identity_pool" "github" {
  workload_identity_pool_id = "github-actions"
  display_name              = "GitHub Actions"
  project                   = var.project_id
}

# OIDC Provider
resource "google_iam_workload_identity_pool_provider" "github" {
  workload_identity_pool_id          = google_iam_workload_identity_pool.github.workload_identity_pool_id
  workload_identity_pool_provider_id = "github-provider"

  attribute_condition = <<-EOT
    assertion.repository_owner_id == "${var.github_owner_id}"
  EOT

  attribute_mapping = {
    "google.subject"       = "assertion.sub"
    "attribute.actor"      = "assertion.actor"
    "attribute.repository" = "assertion.repository"
    "attribute.ref"        = "assertion.ref"
  }

  oidc {
    issuer_uri = "https://token.actions.githubusercontent.com"
  }
}

# Allow GitHub Actions to impersonate Terraform SA
resource "google_service_account_iam_member" "wif" {
  service_account_id = google_service_account.terraform.name
  role               = "roles/iam.workloadIdentityUser"
  member             = "principalSet://iam.googleapis.com/${google_iam_workload_identity_pool.github.name}/attribute.repository/${var.github_org}/${var.github_repo}"
}

output "wif_provider" {
  value = google_iam_workload_identity_pool_provider.github.name
  description = "Full WIF provider path for GitHub Actions auth step"
}

output "terraform_sa_email" {
  value = google_service_account.terraform.email
}
```

### Required Terraform SA Roles by Resource

| Resource Managed               | Minimum Role                          |
|---------------------------------|---------------------------------------|
| Cloud Run services              | `roles/run.admin`                     |
| Service account assignment      | `roles/iam.serviceAccountUser`        |
| Spanner instances/databases     | `roles/spanner.admin`                 |
| Pub/Sub topics/subscriptions    | `roles/pubsub.admin`                  |
| Cloud Storage buckets           | `roles/storage.admin`                 |
| Secret Manager secrets          | `roles/secretmanager.admin`           |
| Monitoring policies             | `roles/monitoring.admin`              |
| VPC Access connectors           | `roles/vpcaccess.admin`               |
| IAM bindings                    | `roles/resourcemanager.projectIamAdmin` |
| Service accounts                | `roles/iam.serviceAccountAdmin`       |
| Workload Identity pools         | `roles/iam.workloadIdentityPoolAdmin` |
| Terraform state bucket          | `roles/storage.objectAdmin` (on bucket) |
