## 1. Client Setup

### Basic Client

```go
import (
    "context"
    "log"

    "cloud.google.com/go/spanner"
)

func main() {
    ctx := context.Background()
    const db = "projects/my-project/instances/my-instance/databases/my-db"

    client, err := spanner.NewClient(ctx, db)
    if err != nil {
        log.Fatal(err)
    }
    defer client.Close()
}
```

### Client with Session Pool Config

```go
client, err := spanner.NewClientWithConfig(ctx, db, spanner.ClientConfig{
    SessionPoolConfig: spanner.SessionPoolConfig{
        MinOpened:           100,  // default 100
        MaxOpened:           400,  // default 400
        TrackSessionHandles: true, // debugging: detect leaks
        InactiveTransactionRemovalOptions: spanner.InactiveTransactionRemovalOptions{
            ActionOnInactiveTransaction: spanner.WarnAndClose,
        },
    },
})
```

### Multiplexed Sessions (v1.85+)

Multiplexed sessions are **enabled by default** in Go client >= v1.85.0. All requests
share a single long-lived session per channel — no need to tune MinOpened/MaxOpened
for read-only workloads.

Environment variables for fine-grained control:

| Variable | Effect |
|---|---|
| `GOOGLE_CLOUD_SPANNER_MULTIPLEXED_SESSIONS=TRUE` | Enable (default in v1.85+) |
| `GOOGLE_CLOUD_SPANNER_MULTIPLEXED_SESSIONS_PARTITIONED_OPS=TRUE` | Enable for partitioned ops |
| `GOOGLE_CLOUD_SPANNER_MULTIPLEXED_SESSIONS_FOR_RW=True` | Enable for read-write txns |

Sessions auto-rotate every 7 days. The regular session pool still backs
read-write transactions unless the RW env var is set.

### Custom Timeout and Retry

```go
import (
    spapi "cloud.google.com/go/spanner/apiv1"
    gax "github.com/googleapis/gax-go/v2"
    "google.golang.org/grpc/codes"
)

co := &spapi.CallOptions{
    ExecuteSql: []gax.CallOption{
        gax.WithRetry(func() gax.Retryer {
            return gax.OnCodes([]codes.Code{codes.Unavailable}, gax.Backoff{
                Initial:    500 * time.Millisecond,
                Max:        16000 * time.Millisecond,
                Multiplier: 1.5,
            })
        }),
    },
}
client, err := spanner.NewClientWithConfig(ctx, db, spanner.ClientConfig{
    CallOptions: co,
})
```

### Always Close

```go
defer client.Close() // returns sessions to pool, closes gRPC channels
```

## 2. Schema Design

### Primary Key Distribution

**Never use sequential/monotonic keys as the first key part.**
Sequential keys (auto-increment, timestamps) create hotspots — all writes land on a
single split.

#### UUIDv4 (recommended for string keys)

```sql
CREATE TABLE Users (
  UserId UUID DEFAULT (NEW_UUID()),
  Email  STRING(320) NOT NULL,
  Name   STRING(256),
) PRIMARY KEY (UserId);
```

Client-generated UUIDs must be lowercase (RFC 4122) to safely mix with
Spanner-generated ones.

#### Bit-Reversed Sequences (recommended for INT64 keys)

```sql
CREATE SEQUENCE UserIdSeq OPTIONS (sequence_kind = "bit_reversed_positive");

CREATE TABLE Users (
  UserId INT64 DEFAULT (GET_NEXT_SEQUENCE_VALUE(SEQUENCE UserIdSeq)),
  Email  STRING(320) NOT NULL,
  Name   STRING(256),
) PRIMARY KEY (UserId);
```

Or with IDENTITY columns:

```sql
CREATE TABLE Users (
  UserId INT64 GENERATED BY DEFAULT AS IDENTITY (BIT_REVERSED_POSITIVE),
  Email  STRING(320) NOT NULL,
  Name   STRING(256),
) PRIMARY KEY (UserId);
```

#### Logical Sharding via Hash

When you must keep a monotonic column (e.g., timestamp), prepend a computed shard:

```sql
CREATE TABLE Events (
  ShardId    INT64 NOT NULL
    AS (MOD(FARM_FINGERPRINT(CAST(CreatedAt AS STRING)), 2048)) STORED,
  CreatedAt  TIMESTAMP NOT NULL,
  EventId    UUID DEFAULT (NEW_UUID()),
  Payload    BYTES(MAX),
) PRIMARY KEY (ShardId, CreatedAt DESC, EventId);
```

Set shard count ~= expected node count. It is static after deployment.

### Interleaved Tables

Interleaving **physically co-locates** child rows with their parent. Use when the
application frequently reads parent + children together (e.g., Order + OrderItems).

```sql
CREATE TABLE Singers (
  SingerId   INT64 NOT NULL,
  FirstName  STRING(1024),
  LastName   STRING(1024),
) PRIMARY KEY (SingerId);

CREATE TABLE Albums (
  SingerId   INT64 NOT NULL,
  AlbumId    INT64 NOT NULL,
  AlbumTitle STRING(MAX),
) PRIMARY KEY (SingerId, AlbumId),
  INTERLEAVE IN PARENT Singers ON DELETE CASCADE;

CREATE TABLE Songs (
  SingerId INT64 NOT NULL,
  AlbumId  INT64 NOT NULL,
  SongId   INT64 NOT NULL,
  SongName STRING(MAX),
) PRIMARY KEY (SingerId, AlbumId, SongId),
  INTERLEAVE IN PARENT Albums ON DELETE CASCADE;
```

**Rules:**

- Child PK must prefix with the **complete** parent PK columns, in order
- Max depth: **7 levels**
- `ON DELETE CASCADE` — delete children when parent is deleted
- `ON DELETE NO ACTION` — block parent deletion if children exist
- Do not use both interleaving and foreign keys for the same relationship

**When to use interleaving vs separate tables:**

| Use interleaving when | Use separate tables when |
|---|---|
| Parent + children always read together | Entities are queried independently |
| Parent-child relationship is 1:N, tightly coupled | Many-to-many relationships |
| Need co-located joins for low latency | Child rows grow very large (GBs per parent) |
| Locality matters for write performance | Relationship is loose / optional |

### DDL Statements in Go

```go
import (
    database "cloud.google.com/go/spanner/admin/database/apiv1"
    adminpb "cloud.google.com/go/spanner/admin/database/apiv1/databasepb"
)

adminClient, err := database.NewDatabaseAdminClient(ctx)
if err != nil {
    return err
}
defer adminClient.Close()

op, err := adminClient.UpdateDatabaseDdl(ctx, &adminpb.UpdateDatabaseDdlRequest{
    Database: db,
    Statements: []string{
        `CREATE TABLE Users (
            UserId UUID DEFAULT (NEW_UUID()),
            Email  STRING(320) NOT NULL,
            Name   STRING(256),
        ) PRIMARY KEY (UserId)`,
        `CREATE INDEX UsersByEmail ON Users(Email)`,
    },
})
if err != nil {
    return err
}
if err := op.Wait(ctx); err != nil {
    return err // DDL failed
}
```

## 3. Read/Write Transactions

### Basic Pattern

```go
commitTs, err := client.ReadWriteTransaction(ctx,
    func(ctx context.Context, txn *spanner.ReadWriteTransaction) error {
        // Read
        row, err := txn.ReadRow(ctx, "Accounts", spanner.Key{"alice"}, []string{"Balance"})
        if err != nil {
            return err
        }
        var balance int64
        if err := row.Column(0, &balance); err != nil {
            return err
        }

        if balance < 100 {
            return fmt.Errorf("insufficient funds: %d", balance)
        }

        // Write (buffered until commit)
        return txn.BufferWrite([]*spanner.Mutation{
            spanner.Update("Accounts",
                []string{"AccountId", "Balance"},
                []interface{}{"alice", balance - 100}),
        })
    },
)
```

### Automatic Retries

The client library **automatically retries** on `ABORTED` errors. The function passed
to `ReadWriteTransaction` may execute multiple times. Consequences:

- **Do not** perform side effects (HTTP calls, Pub/Sub publishes) inside the function
  body. They will repeat on retry.
- Make operations **idempotent** where possible.
- Use `InsertOrUpdate` instead of `Insert` when retries may re-apply the same write.

### Wall-Time Limits

Control total retry duration with context deadlines, not attempt counts:

```go
ctx, cancel := context.WithTimeout(context.Background(), 60*time.Second)
defer cancel()

_, err := client.ReadWriteTransaction(ctx,
    func(ctx context.Context, txn *spanner.ReadWriteTransaction) error {
        // ... transaction body
        return nil
    },
)
// err may be DEADLINE_EXCEEDED if retries exhaust the timeout
```

### Transaction with Options

```go
resp, err := client.ReadWriteTransactionWithOptions(ctx,
    func(ctx context.Context, txn *spanner.ReadWriteTransaction) error {
        // ...
        return nil
    },
    spanner.TransactionOptions{
        CommitOptions:  spanner.CommitOptions{ReturnCommitStats: true},
        TransactionTag: "transfer-funds",
    },
)
if err == nil {
    fmt.Printf("Mutation count: %d\n", resp.CommitStats.MutationCount)
}
```

### Idle Transaction Limits

Spanner aborts transactions idle for >10 seconds (no outstanding reads or queries).
Keep transactions short — read, compute, write, return.

## 4. Mutations vs DML

### client.Apply() — Write-Only (No Reads Needed)

```go
_, err := client.Apply(ctx, []*spanner.Mutation{
    spanner.InsertOrUpdate("Singers",
        []string{"SingerId", "FirstName", "LastName"},
        []interface{}{1, "Marc", "Richards"}),
    spanner.InsertOrUpdate("Singers",
        []string{"SingerId", "FirstName", "LastName"},
        []interface{}{2, "Catalina", "Smith"}),
    spanner.InsertOrUpdate("Albums",
        []string{"SingerId", "AlbumId", "AlbumTitle"},
        []interface{}{1, 1, "Total Junk"}),
})
```

`Apply` creates an implicit transaction, applies mutations, and commits. Use when
you don't need to read data first.

`ApplyAtLeastOnce()` option skips replay protection for higher throughput:

```go
_, err := client.Apply(ctx, mutations, spanner.ApplyAtLeastOnce())
```

### ReadWriteTransaction + DML — Read-Then-Write

```go
_, err := client.ReadWriteTransaction(ctx,
    func(ctx context.Context, txn *spanner.ReadWriteTransaction) error {
        stmt := spanner.Statement{
            SQL: `UPDATE Albums SET MarketingBudget = MarketingBudget * 2
                  WHERE SingerId = @sid AND AlbumId = @aid`,
            Params: map[string]interface{}{"sid": int64(1), "aid": int64(1)},
        }
        rowCount, err := txn.Update(ctx, stmt)
        if err != nil {
            return err
        }
        fmt.Printf("%d row(s) updated\n", rowCount)
        return nil
    },
)
```

### BatchUpdate — Multiple DML Statements

```go
_, err := client.ReadWriteTransaction(ctx,
    func(ctx context.Context, txn *spanner.ReadWriteTransaction) error {
        stmts := []spanner.Statement{
            {SQL: `INSERT INTO Singers (SingerId, FirstName, LastName) VALUES (10, 'Virginia', 'Watson')`},
            {SQL: `INSERT INTO Singers (SingerId, FirstName, LastName) VALUES (11, 'Peter', 'Parker')`},
        }
        rowCounts, err := txn.BatchUpdate(ctx, stmts)
        if err != nil {
            return err
        }
        for i, c := range rowCounts {
            fmt.Printf("Statement %d affected %d rows\n", i, c)
        }
        return nil
    },
)
```

### Key Differences

| Aspect | Mutations (`Apply`/`BufferWrite`) | DML (`Update`/`BatchUpdate`) |
|---|---|---|
| Visibility within txn | **Not visible** until commit (client-buffered) | **Visible** to subsequent reads in same txn |
| Constraint checking | At commit time | After each statement |
| Syntax | Go API (`spanner.Insert(...)`) | SQL strings |
| Read-your-writes | No | Yes |
| Best for | Blind writes, bulk loads | Conditional updates, read-then-write |

### Never Mix Both

Do **not** use DML and mutations in the same transaction. If both are present,
Spanner executes DML first, then mutations — leading to confusing ordering. Mutations
buffered via `BufferWrite` are invisible to DML statements in the same transaction.

Pick one approach per transaction.

## 5. Stale Reads

### Strong Reads (Default)

Returns the latest committed data. Required for read-write consistency.

```go
iter := client.Single().Query(ctx, stmt) // implicitly strong
defer iter.Stop()
```

### ExactStaleness

Read data as of N seconds ago. Fastest stale option (no timestamp negotiation).
Use for dashboards, analytics, caches.

```go
ro := client.ReadOnlyTransaction().WithTimestampBound(
    spanner.ExactStaleness(15 * time.Second),
)
defer ro.Close()

iter := ro.Read(ctx, "Albums", spanner.AllKeys(),
    []string{"SingerId", "AlbumId", "AlbumTitle"})
defer iter.Stop()
for {
    row, err := iter.Next()
    if err == iterator.Done {
        break
    }
    if err != nil {
        return err
    }
    var singerID, albumID int64
    var title string
    if err := row.Columns(&singerID, &albumID, &title); err != nil {
        return err
    }
}
```

### MaxStaleness

Spanner picks the freshest timestamp within the bound. Slightly fresher results
than ExactStaleness but marginally slower (timestamp negotiation).

**Only works with single-use reads** (not multi-read transactions).

```go
iter := client.Single().
    WithTimestampBound(spanner.MaxStaleness(10 * time.Second)).
    Query(ctx, spanner.NewStatement("SELECT * FROM Users WHERE Active = true"))
defer iter.Stop()
```

### MinReadTimestamp / ReadTimestamp

```go
// Read at an exact wall-clock time
spanner.ReadTimestamp(time.Date(2026, 3, 22, 10, 0, 0, 0, time.UTC))

// Read at any time >= the given timestamp
spanner.MinReadTimestamp(time.Date(2026, 3, 22, 10, 0, 0, 0, time.UTC))
```

### Performance Guidelines

- **Minimum 10 seconds staleness** to get a latency benefit.
- Multi-region deployments benefit most — stale reads can hit the nearest replica.
- Single-region: minimal difference between strong and stale.
- Version GC reclaims old versions after `version_retention_period` (default 1h, max 1w).
  Reading beyond retention fails with `FAILED_PRECONDITION`.

## 6. Change Streams

### DDL Setup

```sql
-- Watch entire database
CREATE CHANGE STREAM EverythingStream FOR ALL
  OPTIONS (retention_period = '7d', value_capture_type = 'OLD_AND_NEW_VALUES');

-- Watch specific tables
CREATE CHANGE STREAM OrderStream FOR Orders, OrderItems
  OPTIONS (retention_period = '3d');

-- Watch specific columns
CREATE CHANGE STREAM PriceStream FOR Products(Price, DiscountPrice)
  OPTIONS (value_capture_type = 'NEW_VALUES');

-- Exclude TTL deletes
CREATE CHANGE STREAM AuditStream FOR ALL
  OPTIONS (
    retention_period = '7d',
    exclude_ttl_deletes = true,
    exclude_delete = false
  );
```

**Value capture types:**

| Type | What is captured |
|---|---|
| `OLD_AND_NEW_VALUES` (default) | Old + new values of modified columns |
| `NEW_VALUES` | New values only |
| `NEW_ROW` | All column values when any watched column changes |
| `NEW_ROW_AND_OLD_VALUES` | All new values + old values of modified columns |

**Retention:** 1 to 30 days (default 1 day).

### Manage

```sql
ALTER CHANGE STREAM OrderStream SET FOR Orders, OrderItems, Shipments;
ALTER CHANGE STREAM OrderStream SET OPTIONS (retention_period = '14d');
DROP CHANGE STREAM OrderStream;
```

### Go Consumer — spream (toga4/spream)

Community library for subscribing to change streams directly (without Dataflow).
At-least-once delivery. Single-process only.

```go
import "github.com/toga4/spream"

subscriber, err := spream.NewSubscriber(&spream.Config{
    SpannerClient:    spannerClient,
    StreamName:       "OrderStream",
    PartitionStorage: partitionStorage, // implements spream.PartitionStorage
    Consumer:         &myConsumer{},     // implements spream.Consumer
})
if err != nil {
    return err
}

// Blocks until context is cancelled or error
if err := subscriber.Subscribe(ctx); err != nil {
    log.Printf("subscriber error: %v", err)
}
```

### Go Consumer — spanner-change-streams-tail


## 7. Testing

### Option A: spannertest (In-Process Fake)

Lightweight, no Docker required. Good for unit tests with basic SQL.
**Limited maintenance mode** — missing NUMERIC, JSON, STRUCT types, FOREIGN KEY,
generated columns, and many functions.

```go
import (
    "testing"

    "cloud.google.com/go/spanner"
    "cloud.google.com/go/spanner/spannertest"
    "cloud.google.com/go/spanner/spansql"
    "google.golang.org/api/option"
    "google.golang.org/grpc"
    "google.golang.org/grpc/credentials/insecure"
)

func TestWithSpannertest(t *testing.T) {
    srv, err := spannertest.NewServer("localhost:0")
    if err != nil {
        t.Fatal(err)
    }
    defer srv.Close()
    srv.SetLogger(t)

    // Apply schema
    ddl, err := spansql.ParseDDL("",
        `CREATE TABLE Users (
            UserId STRING(36) NOT NULL,
            Name   STRING(256),
        ) PRIMARY KEY (UserId)`)
    if err != nil {
        t.Fatal(err)
    }
    if err := srv.UpdateDDL(ddl); err != nil {
        t.Fatal(err)
    }

    // Connect via gRPC
    ctx := context.Background()
    conn, err := grpc.DialContext(ctx, srv.Addr,
        grpc.WithTransportCredentials(insecure.NewCredentials()))
    if err != nil {
        t.Fatal(err)
    }
    defer conn.Close()

    db := "projects/test/instances/test/databases/test"
    client, err := spanner.NewClient(ctx, db, option.WithGRPCConn(conn))
    if err != nil {
        t.Fatal(err)
    }
    defer client.Close()

    // Use client normally
    _, err = client.Apply(ctx, []*spanner.Mutation{
        spanner.InsertOrUpdate("Users",
            []string{"UserId", "Name"},
            []interface{}{"abc-123", "Alice"}),
    })
    if err != nil {
        t.Fatal(err)
    }
}
```

Alternative: set `SPANNER_EMULATOR_HOST` instead of using `option.WithGRPCConn`:

```go
os.Setenv("SPANNER_EMULATOR_HOST", srv.Addr)
client, err := spanner.NewClient(ctx, db)
```

### Option B: testcontainers-go (Docker Emulator)

Full-fidelity emulator. Supports all SQL features, constraints, transactions.
Requires Docker.

```go
import (
    "testing"

    "cloud.google.com/go/spanner"
    database "cloud.google.com/go/spanner/admin/database/apiv1"
    adminpb "cloud.google.com/go/spanner/admin/database/apiv1/databasepb"
    instance "cloud.google.com/go/spanner/admin/instance/apiv1"
    instancepb "cloud.google.com/go/spanner/admin/instance/apiv1/instancepb"
    "github.com/testcontainers/testcontainers-go"
    tcspanner "github.com/testcontainers/testcontainers-go/modules/gcloud/spanner"
    "google.golang.org/api/option"
    "google.golang.org/grpc"
    "google.golang.org/grpc/credentials/insecure"
    "google.golang.org/api/option/internaloption"
)

var testClient *spanner.Client

func TestMain(m *testing.M) {
    ctx := context.Background()

    // Start emulator container
    container, err := tcspanner.Run(ctx,
        "gcr.io/cloud-spanner-emulator/emulator:latest",
        tcspanner.WithProjectID("test-project"),
    )
    if err != nil {
        log.Fatal(err)
    }
    defer testcontainers.TerminateContainer(container)

    // Build client options pointing at the emulator
    opts := []option.ClientOption{
        option.WithEndpoint(container.URI()),
        option.WithGRPCDialOption(grpc.WithTransportCredentials(insecure.NewCredentials())),
        option.WithoutAuthentication(),
        internaloption.SkipDialSettingsValidation(),
    }

    projectID := container.ProjectID()
    instanceID := "test-instance"
    databaseID := "test-db"
    dbPath := fmt.Sprintf("projects/%s/instances/%s/databases/%s",
        projectID, instanceID, databaseID)

    // Create instance
    instanceAdmin, _ := instance.NewInstanceAdminClient(ctx, opts...)
    defer instanceAdmin.Close()
    iop, _ := instanceAdmin.CreateInstance(ctx, &instancepb.CreateInstanceRequest{
        Parent:     fmt.Sprintf("projects/%s", projectID),
        InstanceId: instanceID,
        Instance: &instancepb.Instance{
            DisplayName: instanceID,
            Config:      fmt.Sprintf("projects/%s/instanceConfigs/emulator-config", projectID),
            NodeCount:   1,
        },
    })
    if _, err := iop.Wait(ctx); err != nil {
        log.Fatal(err)
    }

    // Create database with schema
    dbAdmin, _ := database.NewDatabaseAdminClient(ctx, opts...)
    defer dbAdmin.Close()
    dop, _ := dbAdmin.CreateDatabase(ctx, &adminpb.CreateDatabaseRequest{
        Parent:          fmt.Sprintf("projects/%s/instances/%s", projectID, instanceID),
        CreateStatement: fmt.Sprintf("CREATE DATABASE `%s`", databaseID),
        ExtraStatements: []string{
            `CREATE TABLE Users (
                UserId STRING(36) NOT NULL,
                Name   STRING(256),
            ) PRIMARY KEY (UserId)`,
        },
    })
    if _, err := dop.Wait(ctx); err != nil {
        log.Fatal(err)
    }

    // Create Spanner client
    testClient, _ = spanner.NewClient(ctx, dbPath, opts...)
    defer testClient.Close()

    os.Exit(m.Run())
}

func TestInsertUser(t *testing.T) {
    ctx := context.Background()
    _, err := testClient.Apply(ctx, []*spanner.Mutation{
        spanner.InsertOrUpdate("Users",
            []string{"UserId", "Name"},
            []interface{}{"u1", "Alice"}),
    })
    if err != nil {
        t.Fatal(err)
    }

    row, err := testClient.Single().ReadRow(ctx, "Users",
        spanner.Key{"u1"}, []string{"Name"})
    if err != nil {
        t.Fatal(err)
    }
    var name string
    if err := row.Column(0, &name); err != nil {
        t.Fatal(err)
    }
    if name != "Alice" {
        t.Fatalf("got %q, want Alice", name)
    }
}
```

### Comparison

| Feature | spannertest | testcontainers (emulator) |
|---|---|---|
| Speed | Fast (in-process) | Slower (container startup) |
| Fidelity | Partial SQL support | Full emulator |
| Docker required | No | Yes |
| JSON/NUMERIC types | No | Yes |
| Foreign keys | No | Yes |
| Transactions | Limited | Full |
| Best for | Unit tests, simple queries | Integration tests, CI |

```hcl
resource "google_spanner_instance" "main" {
  name             = "my-instance"

  display_name     = "Production Instance"
  num_nodes        = 3

```

### google_spanner_database

```hcl
resource "google_spanner_database" "main" {
  instance = google_spanner_instance.main.name
  name     = "my-database"

  # deletion_protection defaults to true — must set false to allow `terraform destroy`
  deletion_protection = true

  version_retention_period = "1h"  # default 1h, max 1w

  ddl = [
    <<-EOT
      CREATE TABLE Users (
        UserId UUID DEFAULT (NEW_UUID()),
        Email  STRING(320) NOT NULL,
        Name   STRING(256),
      ) PRIMARY KEY (UserId)
    EOT
    ,
    "CREATE INDEX UsersByEmail ON Users(Email)",
    <<-EOT
      CREATE TABLE Orders (
        UserId  UUID NOT NULL,
        OrderId UUID DEFAULT (NEW_UUID()),
        Total   FLOAT64,
      ) PRIMARY KEY (UserId, OrderId),
        INTERLEAVE IN PARENT Users ON DELETE CASCADE
    EOT
    ,
    <<-EOT
      CREATE CHANGE STREAM OrderStream FOR Orders
        OPTIONS (retention_period = '7d', value_capture_type = 'NEW_VALUES')
    EOT
    ,

```

**Note for PostgreSQL-dialect databases:** Terraform does not support submitting DDL
while creating a database. DDL must be applied separately.

## 9. Common Patterns

### Batch Inserts with Apply

```go
const batchSize = 500
mutations := make([]*spanner.Mutation, 0, batchSize)

for i, record := range records {
    mutations = append(mutations,
        spanner.InsertOrUpdate("Events",
            []string{"EventId", "Payload", "CreatedAt"},
            []interface{}{record.ID, record.Payload, spanner.CommitTimestamp}),
    )
    if len(mutations) >= batchSize || i == len(records)-1 {
        if _, err := client.Apply(ctx, mutations, spanner.ApplyAtLeastOnce()); err != nil {
            return err
        }
        mutations = mutations[:0]
    }
}
```

### BatchWrite (Mutation Groups)

Non-atomic, higher throughput. Each mutation group commits independently.
**Not replay-protected** — avoid non-idempotent mutations.

```go
mgs := make([]*spanner.MutationGroup, 0, len(records))
for _, r := range records {
    mgs = append(mgs, &spanner.MutationGroup{
        Mutations: []*spanner.Mutation{
            spanner.InsertOrUpdate("Events",
                []string{"EventId", "Payload"},
                []interface{}{r.ID, r.Payload}),
        },
    })
}

iter := client.BatchWrite(ctx, mgs)
if err := iter.Do(func(resp *sppb.BatchWriteResponse) error {
    if resp.Status.Code != 0 {
        log.Printf("group %v failed: %s", resp.Indexes, resp.Status.Message)
    }
    return nil
}); err != nil {
    return err
}
```

### Pagination

#### Key-Based Cursor (Recommended)

Avoids re-reading skipped rows. Spanner treats key conditions as seekable.

```go
func listUsers(ctx context.Context, client *spanner.Client, pageSize int, lastID string) ([]*User, error) {
    stmt := spanner.Statement{
        SQL: `SELECT UserId, Name, Email FROM Users
              WHERE UserId > @lastId
              ORDER BY UserId ASC
              LIMIT @pageSize`,
        Params: map[string]interface{}{
            "lastId":   lastID,
            "pageSize": int64(pageSize),
        },
    }
    iter := client.Single().Query(ctx, stmt)
    defer iter.Stop()

    var users []*User
    for {
        row, err := iter.Next()
        if err == iterator.Done {
            break
        }
        if err != nil {
            return nil, err
        }
        u := &User{}
        if err := row.Columns(&u.ID, &u.Name, &u.Email); err != nil {
            return nil, err
        }
        users = append(users, u)
    }
    return users, nil
}
```

For composite keys or non-unique sort columns, add tie-breaker columns:

```sql
SELECT * FROM Events
WHERE (CreatedAt < @lastTs OR (CreatedAt = @lastTs AND EventId > @lastEventId))
ORDER BY CreatedAt DESC, EventId ASC
LIMIT @pageSize
```

#### LIMIT/OFFSET (Simple, But Costly)

Re-reads and discards all rows before the offset on every page. Avoid for large
result sets or deep pages.

```go
stmt := spanner.Statement{
    SQL:    `SELECT UserId, Name FROM Users ORDER BY Name LIMIT @limit OFFSET @offset`,
    Params: map[string]interface{}{"limit": int64(50), "offset": int64(100)},
}
```

### STRUCT Parameters

Pass Go structs as query parameters. Field names map to STRUCT field names
(or use `spanner:"name"` tags).

```go
type NameFilter struct {
    FirstName string
    LastName  string
}
filter := NameFilter{FirstName: "Elena", LastName: "Campbell"}

stmt := spanner.Statement{
    SQL: `SELECT SingerId FROM Singers
          WHERE (FirstName, LastName) = @name`,
    Params: map[string]interface{}{"name": filter},
}
```

#### Array of STRUCTs with UNNEST

```go
type NameFilter struct {
    FirstName string
    LastName  string
}
names := []NameFilter{
    {"Elena", "Campbell"},
    {"Gabriel", "Wright"},
}

stmt := spanner.Statement{
    SQL: `SELECT SingerId FROM Singers
          WHERE STRUCT<FirstName STRING, LastName STRING>(FirstName, LastName)
          IN UNNEST(@names)`,
    Params: map[string]interface{}{"names": names},
}
```

#### Nested STRUCT Field Access

```go
type SongInfo struct {
    SongName    string
    ArtistNames []NameFilter
}
info := SongInfo{
    SongName:    "Imagination",
    ArtistNames: []NameFilter{{"Elena", "Campbell"}, {"Hannah", "Harris"}},
}

stmt := spanner.Statement{
    SQL: `SELECT SingerId, @songinfo.SongName FROM Singers
          WHERE STRUCT<FirstName STRING, LastName STRING>(FirstName, LastName)
          IN UNNEST(@songinfo.ArtistNames)`,
    Params: map[string]interface{}{"songinfo": info},
}
```

### Query Stats

#### QueryWithStats — Rows + Plan + Execution Statistics

```go
iter := client.Single().QueryWithStats(ctx, spanner.NewStatement(
    "SELECT SingerId, FirstName FROM Singers WHERE LastName = 'Smith'"))
defer iter.Stop()

for {
    row, err := iter.Next()
    if err == iterator.Done {
        break
    }
    if err != nil {
        return err
    }
    // process row...
}

// After iterating, access query stats
if iter.QueryPlan != nil {
    fmt.Printf("Query plan nodes: %d\n", len(iter.QueryPlan.PlanNodes))
}
if iter.QueryStats != nil {
    // iter.QueryStats is map[string]interface{}
    fmt.Printf("Rows returned: %s\n", iter.QueryStats["rows_returned"])
    fmt.Printf("Elapsed time: %s\n", iter.QueryStats["elapsed_time"])
    fmt.Printf("CPU time: %s\n", iter.QueryStats["cpu_time"])
}
```

#### AnalyzeQuery — Plan Only (No Execution)

```go
plan, err := client.Single().AnalyzeQuery(ctx, spanner.NewStatement(
    "SELECT * FROM Users WHERE Email = @email"))
if err != nil {
    return err
}
for _, node := range plan.PlanNodes {
    fmt.Printf("Node %d: %s\n", node.Index, node.DisplayName)
}
```

#### System Query Statistics Tables

```sql
-- Top queries by CPU in the last minute
SELECT text, execution_count, avg_cpu_seconds, avg_latency_seconds, avg_rows_scanned
FROM SPANNER_SYS.QUERY_STATS_TOP_MINUTE

```

### Delete Mutations

```go
// Single key
spanner.Delete("Users", spanner.Key{"user-123"})

// Key range
spanner.Delete("Users", spanner.KeyRange{
    Start: spanner.Key{"a"},
    End:   spanner.Key{"m"},
    Kind:  spanner.ClosedOpen,
})

// All rows
spanner.Delete("Users", spanner.AllKeys())
```

### Struct-Based Mutations

```go
type Singer struct {
    SingerId  int64  `spanner:"SingerId"`
    FirstName string `spanner:"FirstName"`
    LastName  string `spanner:"LastName"`
}

s := Singer{SingerId: 1, FirstName: "Marc", LastName: "Richards"}
m, err := spanner.InsertOrUpdateStruct("Singers", s)
if err != nil {
    return err
}
_, err = client.Apply(ctx, []*spanner.Mutation{m})
```

### Commit Timestamps

Use `spanner.CommitTimestamp` as a value to let Spanner fill in the commit time:

```go
spanner.InsertOrUpdate("Events",
    []string{"EventId", "CreatedAt"},
    []interface{}{"evt-1", spanner.CommitTimestamp})
```

Column must be declared with `OPTIONS (allow_commit_timestamp = true)` in DDL:

```sql
CREATE TABLE Events (
  EventId   STRING(36) NOT NULL,
  CreatedAt TIMESTAMP NOT NULL OPTIONS (allow_commit_timestamp = true),
) PRIMARY KEY (EventId);
```
